# Result Screen & Ladder Ban Recognition Analysis

이 문서는 DJMAX RESPECT V의 **래더 밴(Ladder Ban) 화면** 및 **결과창(Result Screen) 3종**에 대한 레이아웃 분석과 이를 자동 인식하기 위한 기술 설계를 정리한다.

---

## 1. 래더 밴 화면 (Ladder Match Ban & Pick)

### 목적
* 래더 매칭 밴/픽 단계에서 화면에 제시된 5개의 곡 카드를 인식하여, 각 곡의 난이도별 상세 정보(버튼별 서열표 서브넷, V-Archive 개인 최고 기록 등)를 오버레이에 요약 표시한다.

### 레이아웃 특징 및 ROI 구조 (5카드 가로 배치)
* **구조**:
  * 5개의 곡 카드가 가로로 등간격 배치된다.
  * 각 카드에는 **상단 곡명/장르**, **중앙 정사각형 자켓**, **하단 버튼/난이도 뱃지 및 기존 Rate 기록**이 표시된다.
  * 밴 처리(선택)된 카드는 중앙 자켓 위에 붉은색 큰 `"X"` 자 형태의 오버레이가 그려지므로, 픽셀 색상 대조를 통해 밴된 곡인지 판별 가능하다.
* **씬 판별 키워드 (OCR)**:
  * 좌측 상단 로고: `"ONLINE LADDER MATCH"`
  * 상단 메시지 영역: 언어별 메시지 또는 로고 키워드 판독
* **데이터 추출 위치 (ROI)**:
  * 5개 카드 각각의 `jacket_1` ~ `jacket_5` 가로 좌표 영역 (1920x1080 기준 등간격).
  * 5개 카드 각각의 `button_diff_1` ~ `button_diff_5` 영역을 통해 개별 차트 정보 파싱.

## 2. 래더 매칭 대기 / 라운드 시작 화면 (Ladder Match Round)

### 목적
* 매칭 완료 후, 매 라운드(Round 1~3) 시작 시 중앙에 공개되는 곡 정보를 감지하여 해당 라운드 진행 곡의 메타 정보를 즉시 오버레이에 렌더링한다.

### 레이아웃 특징 및 ROI 구조 (중앙 세로 배치)
* **구조**:
  * 화면 중앙 세로 열에 3개의 라운드 슬롯이 수직 배치된다.
  * 활성화된 현재 라운드 슬롯(예: Round 1)에는 **곡 제목, 190 BPM 등 정보, 정사각형 자켓, 8B MAXIMUM 등 배너**가 공개된다. 아직 치르지 않은 라운드 슬롯은 물음표(`?`) 아이콘으로 숨겨져 있다.
* **씬 판별 키워드 (OCR)**:
  * 중앙 배경의 거대 텍스트 `"ROUND"` 또는 상단 라운드 번호
* **데이터 추출 위치 (ROI)**:
  * 중앙 활성 라운드 슬롯의 `jacket` 및 `title` 영역.

## 2. 결과창 3종 (Result Screens)

### 목적
* 플레이 완료 직후 결과창에서 최종 판정(Rate, Max Combo 등)과 곡 정보를 파악해 로컬 SQLite DB(`record.db`)에 누수 없이 실시간으로 자동 갱신/저장한다.
* 결과창은 모드(프리스타일 / 오픈매치 인원수)에 따라 총 3종의 레이아웃으로 세분화된다.

### 상세 분석 및 구분 기준

```
┌─────────────────────────┬──────────────────────────────────┬─────────────────────────────────┐
│  프리스타일 결과창       │   오픈매치 (1인 / 3인 이상)       │        오픈매치 (2인)           │
│      [Layout A]         │            [Layout B]            │           [Layout C]            │
├─────────────────────────┼──────────────────────────────────┼─────────────────────────────────┤
│                         │                                  │                                 │
│  ┌──────────┐  ┌─────┐  │  ┌──────────┐  ┌───┐┌───┐┌───┐   │  ┌──────────┐   ┌──────────┐    │
│  │ Judgement│  │Play │  │  │  Local   │  │P2 ││P3 ││P4 │   │  │  Local   │   │ Opponent │    │
│  │ Details  │  │Point│  │  │  Player  │  └───┘└───┘└───┘   │  │  Player  │   │  Player  │    │
│  │          │  │     │  │  │  Circle  │  ┌───┐┌───┐┌───┐   │  │  Circle  │   │  Circle  │    │
│  │ (Left)   │  │(Rgt)│  │  │          │  │P5 ││P6 ││   │   │  │          │   │          │    │
│  └──────────┘  └─────┘  │  └──────────┘  └───┘└───┘└───┘   │  └──────────┘   └──────────┘    │
│                         │                                  │  (DJ POWER O)   (DJ POWER X)    │
│                         │                                  │                                 │
└─────────────────────────┴──────────────────────────────────┴─────────────────────────────────┘
```

### 1) 프리스타일 결과창 (Layout A)
* **씬 감지 앵커 (OCR)**:
  * 좌측 패널 상단: `"JUDGEMENT DETAILS"`
  * 우측 상단 패널: `"PLAY POINT"`
  * 우측 하단 버튼 안내 가이드: `F5` (레더보드), `F8` (즐겨찾기), `Enter` (확인)
* **데이터 추출 위치 (ROI)**:
  * **자켓 (곡 식별)**: 최상단 중앙-좌측 (`Nightmare` 등 제목 텍스트 왼쪽)
  * **버튼/난이도**: 최상단 중앙 배너 (`6 BUTTON TUNES` 및 `HARD` 등 난이도)
  * **판정 (Rate)**: 중앙 큰 원 하단에 위치한 판정율 백분율 (`99.08%`)

### 2) 오픈매치 1인 / 3인 이상 결과창 (Layout B)
* **씬 감지 앵커 (OCR)**:
  * 우측 그리드에 다수의 플레이어 미니 판정 패널 존재
  * 우측 하단 버튼 안내 가이드: `Space` (상세정보), `F8` (즐겨찾기), `Enter` (확인) -> 프리스타일의 `F5`가 `Space`로 변경됨
* **데이터 추출 위치 (ROI)**:
  * **자켓 (곡 식별)**: 최상단 중앙 (`Sunny Side` 등 제목 텍스트 왼쪽)
  * **버튼/난이도**: 로컬 플레이어 판정 서클 하단 뱃지 (`4B TUNES SC` 녹색 배너)
  * **판정 (Rate)**: 로컬 플레이어 판정 서클 바로 아래의 백분율 (`99.78%`)

### 3) 오픈매치 2인 결과창 (Layout C)
* **씬 감지 앵커 (OCR)**:
  * 좌우 대칭형 2분할 레이아웃
  * 판정 서클 주변에 대각선 번개/빛 연출 및 `"WINNER"` / `"LOSER"` 뱃지
* **로컬 플레이어 식별 로직**:
  * 로컬 플레이어의 서클 영역 아래에만 **`DJ POWER` 점수 및 `NEW RECORD` 주황색 배너**가 표시됨 (상대방은 해당 위치가 공란).
* **데이터 추출 위치 (ROI)**:
  * **자켓 (곡 식별)**: 최상단 중앙 (`Break!` 등 제목 텍스트 왼쪽)
  * **버튼/난이도**: 로컬 플레이어 서클 하단의 모드/난이도 뱃지

## 3. MAX COMBO / PERFECT PLAY 뱃지 인식 방안

선곡창에서의 Max Combo 인식 방식(특정 위치 뱃지의 평균 밝기 체크)과 달리, 결과창에서는 다음과 같은 최적의 대안들을 조합하여 판별한다.

### ① BREAK 횟수 기반 간접 판정 (가장 추천 및 통합 적용 가능)
* **원리**: 프리스타일, 오픈매치 1인/3인, 오픈매치 2인 결과창 모두 로컬 플레이어 판정 서클의 **우측 상단에 `[숫자] BREAK` 형태의 텍스트**가 고정 노출된다.
* **로직**:
  * 해당 영역을 OCR로 읽어 `BREAK` 숫자가 `0`으로 파싱되면 **MAX COMBO**로 판정한다.
  * `BREAK`가 `0`이고 동시에 최종 판정 `Rate`가 `100.00%`이면 **PERFECT PLAY**로 판정한다.
* **장점**: 모든 결과창 레이아웃에 공통적으로 적용 가능하여 ROI 관리 및 예외 처리 비용이 대폭 절감된다.

### ② 리본/배너 색상 픽셀 매칭 (보완책)
* **원리**: MAX COMBO 달성 시 판정 서클 우측 하단에 **하늘색/민트색 대각선 리본**이 추가되며, PERFECT PLAY 달성 시 **무지개색 그라데이션 배너**가 서클을 가로지른다.
* **로직**:
  * 리본이 위치하는 특정 ROI 영역의 평균 색상 또는 특정 색상(하늘색 `HSV`/`RGB` 값)의 픽셀 점유율을 측정한다.
  * 평균 밝기만 확인하던 선곡창 방식과 달리, 특정 대표 색상 범위와의 Euclidean 거리를 계산하여 매칭 신뢰도를 확보한다.

## 3. 구현 단계별 로드맵 (Proposed Roadmap)

### Phase 1: SceneType 확장 및 OCR 씬 판별 기초 인프라 구축
1. **SceneType 추가**:
   * `SceneType::LadderBan`
   * `SceneType::ResultFreestyle`
   * `SceneType::ResultOpenMatch`
   * `SceneType::ResultOpenMatch2P`
2. **OCR 엔진 키워드 룰셋 보강**:
   * [ocr_engine.rs](../rust/overmax_app/src/ocr_engine.rs) 내에 결과창 및 래더 밴 텍스트 매칭 기준 고도화.
   * `match_logo_scene` 함수에 해당 화면 진입 확인 앵커 추가.

### Phase 2: ROI 데이터 설계 및 결과창 기록 수집 연동
1. **DPI 대응 ROI 좌표 등록**:
   * `settings.json` 설정 모델에 3가지 결과창 씬별 자켓, 난이도 뱃지, 판정(Rate) OCR ROI 영역 추가.
2. **결과창 디텍션 및 DB 자동 저장**:
   * 결과창 상태 진입 감지 시, 자켓 인식으로 곡 ID를 확보하고 Rate 값을 Windows OCR로 읽어 안정적 상태 판정 후 `record.db`에 upsert 수행.

### Phase 3: 래더 밴 화면 멀티 자켓 매칭 오버레이 바인딩
1. **Jacket Tracker 멀티플렉싱**:
   * 디텍션 루프에서 래더 밴 씬 감지 시 5개 카드 영역의 자켓을 추출하여 순차적으로 HOG/해시 매칭 수행.
2. **오버레이 뷰 정보 표시**:
   * 5개 곡의 서열표 정보 등을 가로 또는 세로 리스트 스킨으로 오버레이에 렌더링.
