# Overmax v0.2.4 릴리즈 노트

> v0.2.3 이후 변경 사항

---

## 🚀 주요 신규 기능 및 변경 사항

### 📊 결과 화면(Result Scene) 인식 도입 및 실시간 최고 기록 피드백 UI (가장 큰 작업)
* **결과 화면(Result Scene) 인식 및 씬 확장**:
  * FREESTYLE 및 ONLINE 대기방 외에도 플레이 완료 후 나타나는 결과 화면 씬(`ResultFreestyle`, `ResultOpen2`, `ResultOpen3`)을 신규 정의하고 인식 파이프라인에 통합했습니다.
  * 결과창 진입 시 Hysteresis 감쇄 판정 오동작을 수정하여 화면 흔들림 및 인식 끊김 Jitter를 해소하고 안정적으로 데이터를 유지하도록 했습니다.
* **최고 기록 대비 실시간 차이 표기**:
  * 결과 화면 진입 시, 헤더 2열 및 라이트모드 2열의 메타 정보를 숨기고 **기존 최고 기록 대비 현재 실시간 레이트의 변동 수치**(`(99.00% +0.50%)` 등)를 계산하여 동적으로 출력하는 UI 피드백을 추가했습니다.
* **뱃지 인식의 이미지 해시 템플릿 매칭 전환**:
  * 결과창 및 선곡창의 Max Combo / Perfect Play 뱃지 감지 로직을 기존의 불안정한 픽셀 검사 방식에서 이미지 해시(pHash, dHash, aHash) 기반 템플릿 매칭 방식으로 전면 교체하여, 연출 그래픽 변화나 프레임 흐려짐에 따른 오인식을 차단했습니다.
* **소수점 반올림 및 절사 보정 오류 해결**:
  * 결과창 Rate OCR 도중 소수점 셋째 자리 `9`가 반올림되면서 실제 수치와 어긋나게 감지되던 버그를 정밀 절사/버림 보정 로직을 추가하여 해결했습니다.
* **인식 파이프라인 구조 단순화 (bottom_guide 제거)**:
  * 결과창 내에서 앵커(기준점) 역할을 수행하던 레거시 `bottom_guide` 감지 모듈을 전면 걷어내고, 뱃지 OCR 텍스트 감지 결과 자체를 앵커로 활용하도록 통일하여 결과 화면 인식 구조의 불필요한 의존성을 대폭 단순화했습니다.

### 🎯 인식 엔진 고도화 및 정확도 개선 (OCR & 이미지 매칭)
* **Rate/Score 템플릿 매칭 오인식(8 ↔ 3) 해결**:
  * 픽셀 노이즈로 인해 숫자 '8'이 '3'으로 잘못 매칭되던 오독 결함을 수정했습니다.
  * 템플릿 점수 타이(Tie) 발생 시 '8'이 먼저 평가되도록 배열 상수 정렬 순서를 재정비하고, '8' 템플릿에 형태 포함 관계를 감안한 미세 가중치(`+0.005`, 약 0.5% 기여)를 반영하여 8의 왼쪽 세로 기둥 유실 시에도 3으로 잘못 매칭되는 문제를 방어했습니다.
* **선곡창 및 결과창 Rate OCR 캐시 비활성화 (DB 오염 방지)**:
  * 동일 곡/난이도 메타데이터 조건에서 OCR을 스킵하던 기존 캐싱 가드를 걷어내고, **항상 200ms 주기로 Rate OCR을 재검사**하도록 교정했습니다.
  * 동일 곡 내부에서 기록 탭을 전환하거나 다른 플레이어의 결과창을 볼 때 수치 변화가 실시간으로 반영되어 예전 수치 고착으로 인해 로컬 SQLite 데이터베이스가 오염되는 현상을 예방했습니다.
* **헤더 2열 UI 레이아웃 폰트 크기 일괄 동기화**:
  * 일반 모드 오버레이 헤더 2열 내의 플레인 텍스트(괄호 비교문, 선곡창 메타 텍스트)의 폰트 크기를 뱃지 텍스트와 통일성 있게 `10.0 -> 9.0`으로 일괄 축소 정렬하여 시각적 불균형과 이질감을 교정했습니다.
* **OCR 다중 패스 폐기 및 단일 패스(1-Pass) 롤백 (성능 보호)**:
  * 인게임 실시간 분석 스레드에 가해지던 CPU 오버헤드를 막기 위해 3-pass 등의 다중 패스 OCR 루프를 영구히 폐기하고, 단일 패스(1-Pass) OCR 방식으로 단일화했습니다.
  * 다중 패스 제거에 따른 오독 취약점은 HysteresisBuffer 기반의 프레임 다수결 신뢰 안정화 필터링을 통해 정밀하게 보완했습니다.
* **무거운 HOG 자켓 이미지 매칭의 선택적 비활성화 옵션 (`disable_hog`) 지원**:
  * CPU 프로파일링 상에서 점유율의 큰 주범이 될 수 있는 자켓 영역 HOG(Histogram of Oriented Gradients) 매칭을 사용자의 컴퓨터 사양에 맞춰 비활성화할 수 있는 옵션(`disable_hog`, 기본값 `false`)을 제공합니다.
  * HOG를 비활성화하더라도 aHash, dHash, pHash 3종 Perceptual Hash의 유기적인 결합만으로 충분히 뛰어난 자켓 매칭 변별력을 제공하므로, 필요시 CPU 오버헤드를 대폭 완화하는 데 활용할 수 있습니다.

### ⏱️ 오버레이 반응성 개선 및 프레임 Jitter 차단
* **0ms 즉시 숨김 반응성 확보**:
  * 곡 연주 시작이나 씬 이탈 등으로 Hysteresis 신뢰도가 0이 되는 순간, 불필요한 렌더링 딜레이 프레임 대기 없이 즉각적으로 오버레이 창을 숨기도록 처리해 UI 반응 체감을 개선했습니다.

### ⚡ CPU 및 메모리 사용량 단계적 최적화 (성능 개선)
* **GDI 캡처 인플레이스 버퍼 재사용 (memcpy 제거)**:
  * Rust 프로파일러(samply) 종합 분석 결과를 기반으로, 기동 시 매 프레임 발생하던 힙 메모리 재할당 및 대량의 memcpy 오버헤드를 추적하여 제거했습니다.
  * `CaptureEngine::capture_bgra`에 out-parameter 인플레이스 버퍼 재사용 기법을 도입하여 평상시 힙 재할당 횟수를 **0회**로 최적화했습니다.
* **WindowTracker 동적 조절(Dynamic Polling) 도입**:
  * 게임 창 유무와 포커스 상태를 조회하는 Win32 시스템 콜 오버헤드를 막기 위해 창 이동/드래그 중에는 16ms, 평상시 정지 상태에는 300ms, 창 미발견 시 1000ms 주기로 폴링 간격을 자동 조정하게 하여 CPU 스팸을 억제했습니다.
* **AdaptiveCaptureEngine DXGI 재시도 쿨다운 도입**:
  * DXGI Desktop Duplication 캡처 실패로 GDI 백엔드로 폴백할 때, 매 프레임 재기동을 시도하여 발생하던 스팸 연산을 최소 **3초**의 쿨다운 간격을 두어 차단했습니다.
* **변화량 감지 캐시(픽셀 체크섬 조기 리턴) 도입**:
  * 화면 픽셀 변화가 없을 때는 무거운 이미지 크롭 및 매칭 분석을 조기에 바이패스하고 스킵하는 `compute_pixel_checksum` 조기 리턴 로직을 강화했습니다.
* **Rust 데이터 모델 메모리 레이아웃 개선 (힙 해제)**:
  * V-Archive 곡/패턴 데이터의 `HashMap` 구조를 고정 크기 2차원 배열(`[[Option<PatternInfo>; 4]; 4]`)로 전면 전환하여 힙 메모리 할당을 대폭 해제했습니다.
  * `GoldMeta`, `AssistMeta` Enum 타입 경량화 및 중복도가 높은 문자열에 `Arc<str>` 풀링을 적용하여 메모리 잔존 점유율을 극대화했습니다.

### 🛡️ 스팀(Steam) 경로 스캔 폴백 및 안전 예외 가드 도입
* **실행 중인 프로세스 스캔을 통한 스팀 경로 동적 검출**:
  * 스팀 레지스트리가 유실되거나 비표준/포터블로 설치된 경우, Win32 API를 통해 실행 중인 `steam.exe` 프로세스의 이미지 경로를 동적으로 추적하여 V-Archive 계정 연동 설정 파일(`loginusers.vdf`)을 찾아내도록 연동 성능을 보강했습니다.
* **Mutex 락 오염(Poisoning) 복구**:
  * Mutex 락 획득 시 락 오염으로 인해 일어나는 연쇄 크래시(Crash)를 방지하기 위해 오염 감지 시 자동으로 내부 락을 강제 회복하는 `.unwrap_or_else(|e| e.into_inner())` 방어 가드를 이식했습니다.
* **Win32 창 트래킹 및 UI 설정 파일 안전 예외 가드**:
  * 창 트래킹 관련 `.unwrap()` 위주의 불안정한 Win32 호출 코드들을 안전한 Option 매핑 구조로 치환하여 패닉 가능성을 완전 배제했으며, 사용자 UI 설정 파일 파싱 예외 발생 시 안전하게 기본값으로 복구하는 Fallback을 구축했습니다.
