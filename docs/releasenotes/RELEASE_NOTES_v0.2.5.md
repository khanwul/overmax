# Overmax v0.2.5 릴리즈 노트

> v0.2.4 이후 변경 사항

---

## 🚀 주요 신규 기능 및 변경 사항

### 🎯 1. Rate/Score 및 탐지(Detection) 정확도 개선 (템플릿 매칭 고도화)
* **템플릿 매칭 및 오독 해결**:
  * Rate/Score 템플릿 매칭 알고리즘을 개선하여 픽셀 왜곡으로 발생하던 오독(예: '8'과 '3'의 혼동 등)을 해결했습니다.
  * 결과창의 버튼 모드(4B) 인식 임계값을 유연하게 낮춰 인식 누락을 차단하고, 결과창 뱃지 해시 매칭 감도를 정밀하게 조율했습니다.
* **글로벌 명암 기반 이진화 전격 적용**:
  * 선곡창 및 결과창 전반에 BGA(배경 애니메이션) 노이즈 간섭에 강인한 **글로벌 명암 기반 이진화(`binarize_by_global_contrast`)**를 이식하여 텍스트 영역을 선명히 분리하도록 개선했습니다.
* **빈 프레임 오독 방지 가드레일 도입**:
  * 게임 화면 로딩이나 검은 빈 화면(Blank Frame) 진입 시 난이도가 SC로 오인식되어 캐시 복구가 꼬이던 결함을 해결했습니다. 이진화 후의 유효 픽셀(자획) 수가 기준치 미만일 경우 즉시 스킵하는 가드레일을 추가했습니다.
* **인식 실패 시 OCR 폴백 가드 보완**:
  * 스코어 템플릿 매칭 결과의 파싱이 최종 실패할 때, 1-pass 이진화 OCR 폴백을 매끄럽게 태워 감지를 끝까지 성공적으로 복구하도록 예외 구조를 단단히 다듬었습니다.
* **즐겨찾기 뱃지 오염에 따른 자켓 매칭 기본 임계값 조정**:
  * 곡을 즐겨찾기(Favorite)에 등록했을 때 자켓 모퉁이에 생기는 리본 뱃지로 인해 자켓 해시 매칭 스코어가 저하되어 인식이 누락되는 문제를 해결했습니다.
  * 자켓 매칭의 기본 유사도 임계값(`similarity_threshold`)을 `0.75`에서 `0.70`으로 조정하여, `Fundamental` 등 즐겨찾기 가림막이 생긴 곡들의 인식률을 완벽히 보장합니다.

### 🐧 2. 리눅스(Linux) 빌드 공식 지원
* **리눅스 빌드 스텁(Stub) 및 호환성 이식**:
  * 리눅스 환경에서도 Overmax 크레이트들의 빌드 및 컴파일이 정상 지원되도록 크로스 컴파일 호환용 빌드 스텁을 정비하고 컴파일 에러들을 해소했습니다.

### 🛠️ 3. 내부 아키텍처 대대적 리팩토링 및 UI 모듈화
* **UI 패널 컴포넌트 격리 및 커스텀 위젯 분리**:
  * 오버레이 UI 렌더링에 집중되어 있던 코드를 역할별 컴포넌트(`Header`, `PlayMetaRow`, `StatusLamp` 등) 및 전용 커스텀 위젯 단위로 완전히 분리하고 캡슐화하여 코드 유지보수성을 극대화했습니다.
  * 결과창 최고 기록 비교 텍스트 및 상세 비교 포맷(Perfect 'P', Max Combo 'M') 복구 이식을 컴포넌트 구조 내에 완벽하게 정비했습니다.
* **데이터 레이어 분할 및 모듈 의존성 재정비**:
  * `overmax_data` 모듈 하위 영역을 4대 도메인(`config`, `store`, `community`, `service`)으로 명확히 재구조화하여 크레이트 간 결합도를 낮추고 인프라 구조를 최적화했습니다.
