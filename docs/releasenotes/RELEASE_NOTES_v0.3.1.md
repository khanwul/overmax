# Overmax v0.3.1 릴리즈 노트

> v0.3.0 이후 변경 사항 (v0.3.1)

---

## 🚀 주요 신규 기능 및 사용자 변경 사항

### ⚡ 1. V-Archive 동기화 속도 혁신 & 알림 기능 강화
* **증분 동기화(Incremental Sync) 도입으로 시작 로딩 체감 해소**:
  * 앱 시작 시 및 설정창 갱신 시 전체 데이터를 매번 다시 다운로드하지 않고, 타임스탬프(`since` 파라미터) 기반으로 변경된 플레이 기록만 다운로드하여 로딩 속도와 네트워크 부하를 극도로 줄였습니다.
  * 초경량 동기화가 보장됨에 따라 앱 시작 시 자동 동기화 기능이 상시 활성화되도록 개선되었습니다.
* **단일 곡 업로드 고속 최적화**:
  * 플레이 완료 후 단일 곡 업로드 시 전체 캐시 갱신 대신 해당 곡의 기록만 즉시 로컬 캐시에 머지(Merge) 처리하여, 디스크 I/O 병목 및 업로드 렉 현상을 방지했습니다.
* **실시간 TOP 50 랭킹 달성 토스트(Toast) 알림**:
  * 기록 업로드 완료 시 오버레이 UI의 상단 헤더/상세 영역에 V-Archive TOP 50 내 순위(예: `8B TOP 29위 달성!`)를 실시간 토스트 메시지로 3초간 직관적으로 노출해 줍니다.
* **동기화 화면(Sync Window) 가짜 후보 노출 및 항목 꼬임 버그 수정**:
  * 동기화 화면 스캔 시 갱신 필요가 없는 곡들(약 300여 개)이 가짜 동기화 후보(False Positive)로 잘못 노출되던 결함을 DB 단일 대조 쿼리 방식으로 정비하여 완전히 해결했습니다.
  * 동기화 스캔 및 업로드 진행 중 목록 항목 위치(인덱스)가 꼬여 카드 상태가 잘못 변경되던 현상을 곡 고유 식별자(`RecordKey`) 기반 이벤트 전달 구조로 전환하여 수정했습니다.
* **Steam 세션 계정 자동 감지 폴백 보강**:
  * 최신 Steam `loginusers.vdf` 파일에서 `MostRecent` 키가 생략되었을 때 주계정 대신 서브계정이 무작위로 선택되던 버그를 수정했습니다. (`MostRecent` → `AutoLogin` → 최신 타임스탬프 순 4단계 폴백 적용)

### 🎛️ 2. V-Archive 동기화 창 필터(Sync Filter) 시스템 도입
* **다양한 동기화 후보 필터링 조건 지원**:
  * **모드별 필터**: 4B, 5B, 6B, 8B 토글 버튼으로 필요한 버튼 모드만 추출합니다.
  * **난이도별 필터**: NM, HD, MX, SC 토글 버튼으로 특정 난이도 계열만 조망합니다.
  * **단일 레벨 레인지 슬라이더**: 1~15 및 SC1~SC15까지 총 30단계 레벨 범위를 단일 레인지 바의 핸들 2개(Dual Thumb)로 자유롭게 지정합니다.
  * **단일 Rate 레인지 슬라이더**: 0.0% ~ 100.0% 달성율 범위를 단일 레인지 바 핸들 2개로 조절합니다.
  * **맥스콤보 달성만 체크박스**: 내 기록은 맥스콤보지만 V-Archive에는 맥스콤보가 안 되어 있는 갱신 대상만 필터링합니다.
  * **미업로드 제외 체크박스**: V-Archive 미등록 곡을 숨기고 점수 갱신분만 집중 탐색합니다.
* **접이식 Form Group UI 및 프레임 렉 없는 자동 영속화**:
  * 필터 영역을 접고 펼칠 수 있는 접이식 카드 UI와 좌측 라벨 고정 폭(170px) 정렬을 적용하여 수치 변경 시의 레이아웃 흔들림을 차단했습니다.
  * UI 조작 즉시 백그라운드 스레드에서 `settings.user.json`으로 자동 저장이 수행되어 60fps 드래그 시의 프레임 드랍이나 멈칫거림 현상을 원천 방지했습니다.

### 🎯 3. 즐겨찾기(Favorite) 곡 인식률 개선 및 오탐 차단
* **즐겨찾기 뱃지 마스킹 및 인식률 보정**:
  * 곡을 즐겨찾기에 등록했을 때 자켓 우상단에 생성되는 하트/리본 뱃지 그래픽 노이즈로 인해 인식률이 떨어지던 결함을 마스킹 기법 및 연산 알고리즘 개선으로 완벽히 해결했습니다. 즐겨찾기 등록 여부와 관계없이 100% 정밀하게 곡을 인식합니다.
* **카테고리 띠(Category Band) 4px 통계 감지 고도화**:
  * 곡 자켓 우측 카테고리 띠의 감지 영역을 배경 노이즈가 차단되는 4px 폭으로 정밀화하고 수직 단색성 및 채도 정합성 통계 판정 가드를 적용하여, 자켓 외곽선이 흐릿한 곡이나 신규 DLC 곡들도 오탐 없이 신속히 판별합니다.

### 🏎️ 4. OCR-Free 씬 판독 & 결과창 실시간 독립 감지
* **OCR Bypass 파이프라인 정립으로 반응 속도 극대화**:
  * 선곡창, 결과창, 오픈매치 씬 진입 시 무거운 Windows OCR을 기다리지 않고 자켓 및 엣지 매칭을 우선 수행하는 OCR Bypass 파이프라인을 구축하여 오버레이 화면의 전환 반응 속도를 크게 끌어올렸습니다.
* **결과창(Result Scene) 선곡창 캐시 의존성 완전 제거**:
  * 결과창 진입 시 선곡창에서 읽었던 이전 캐시에 의존하지 않고, 결과창 화면 자체의 픽셀 및 자켓 정보만으로 단독 독립 감지를 수행하여 이전 판의 기록이 잘못 남아 오기록되던 정합성 문제를 원천 차단했습니다.
* **결과창 판독 정밀도 강화**:
  * 오픈매치 결과창의 플레이어 패널 엣지 감지 및 프리스타일 결과창의 모드 컬러바(mode_colorbar) 색상/엣지 추가 검증을 이식하여 결과창 인식의 정밀도를 보강했습니다.

### 🐧 5. 리눅스(Linux) 수직 슬라이스 포팅 완료 (Contributed by @KHanwuL)
* **Linux 네이티브 오버레이 지원 기초 완성**:

  * *(리눅스 네이티브 포팅 및 파이프라인 수직 슬라이스 전체 구현은 **@KHanwuL** 님의 소중한 기여로 완성되었습니다. 감사드립니다!)*

## 🛠️ 개발자 및 내부 아키텍처 변경 사항

### ⚡ 1. 초고속 그리드 히스토그램 이미지 매칭 엔진 도입 (HOG 대체)
* **1차 u64 해시 Early Exit + 2차 2x2 분할 그리드 히스토그램 WTA 도입**:
  * CPU 점유 스파이크 및 메모리 오버헤드의 원인이었던 HOG(Histogram of Oriented Gradients) Cosine 유사도 연산 모듈을 100% 소거했습니다.
  * 대신 `1차 u64 해시 Early Exit (Hamming <= 42)`와 `2차 2x2 분할 그리드 히스토그램 L1 벌점 WTA(Winner-Takes-All)` 매칭 알고리즘을 구축하여, 자켓 매칭 연산 속도를 기존 대비 **종합 122배 (순수 매칭 루프 457배)** 고속화했습니다.
* **64x64 정규화 해상도 일치 및 메모리 절감**:
  * DB 빌드(`db_builder`)와 런타임 간 히스토그램 추출 공간을 64x64 정규화 공간(Lanczos3)으로 단일화했습니다. DB 로딩 시 무거운 HOG float BLOB 파싱을 스킵하여 상주 메모리를 **6MB 절감**했으며, 통계 임계치(`similarity_threshold = 0.65`) 보정을 거쳐 **인식률 100.00% / 오탐률 0.00%**를 달성했습니다. (구버전 호환성을 위해 `db_builder` 내 HOG 적재는 유지)

### 💾 2. SQLite 기반 V-Archive DB 내장화 및 SQL LEFT JOIN 동기화 개편
* **SQLite STORED Generated Columns & O(1) 증분 캐시 갱신**:
  * 기존 레거시 JSON 파일 캐시 구조를 SQLite DB(`varchive_records`)로 내장 일화하고 자동 마이그레이션을 적용했습니다.
  * `score`, `max_combo`, `updated_at`, `rating` 필드를 STORED 물리 생성 컬럼 및 복합 인덱스로 구축하여 O(1) 초고속 조회 및 실시간 랭킹 계산이 가능해졌습니다.
  * `RecordManager` O(1) 증분 캐시 갱신(`upsert_varchive_record`)을 통해 단일 곡 업로드 시 전체 DB 재조회 오버헤드를 완전히 소거했습니다.
* **단일 SQL LEFT JOIN 쿼리 기반 동기화 후보 추출**:
  * 동기화 화면(`SyncWindow`)의 갱신 후보 추출 시 `records`와 `varchive_records` 테이블을 단일 `LEFT JOIN` 쿼리로 대조하여 이중 로드 연산을 지우고, 약 300여 개에 달하던 False Positive(가짜 후보) 노출 결함을 소거했습니다.

### ⚙️ 3. Pure Rust Native 컴파일 지원 (`ocr-fallback` Feature Flag)
* **Windows OCR 의존성 피처 분리**:
  * Cargo Feature Flag `ocr-fallback`을 통해 WinRT / Windows OCR 의존성을 선택적으로 분리할 수 있도록 개선했습니다. `--no-default-features` 옵션 지정 시 Windows OS API 의존성이 0%인 Pure Rust Native 엔진으로 컴파일할 수 있습니다.

### 🧩 4. 코드 모듈화, RecordKey 체계 개편 및 모나딕(Monadic) 패턴 정돈
* **RecordKey 기반 비동기 UI 이벤트 전환**:
  * UI 테이블 인덱스(`usize`) 기반 전달 방식을 `RecordKey(song_id, mode, diff)` 및 `is_quick_upload` 플래그 전달 체계로 개편하여, 비동기 스캔 및 업로드 진행 중 인덱스 꼬임 문제를 원천 방지했습니다.
* **ROI 체이닝 및 모나딕 분기 패턴 정돈**:
  * `RoiManager` / `RoiRect`에 `and_then_roi` 모나딕 체이닝 메서드를 도입하고 `ImageRegion` 내에 CV 래퍼 메서드를 캡슐화했습니다.
  * `ocr_engine.rs`, `play_state.rs`, `client.rs`, `recommend.rs` 등에 흩어져 있던 중첩 `if let` 분기들을 모나딕 체인(`and_then`, `unwrap_or`, `let Some = ... else`) 패턴으로 깔끔하게 정돈했습니다.
