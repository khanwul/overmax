# 2026-06-16 Overmax 신규 피드백 및 동기부여 기능 아이디어

이 문서는 게임 플레이 중 시스템 오버헤드를 유발하지 않으면서도, 로컬 플레이 기록 DB(`cache/record.db`) 및 서열표 메타 데이터를 가공하여 유저에게 강력한 동기부여와 시각적 성취감을 제공하기 위한 핵심 기능 아이디어를 정리한다.

---

## 1. 휴리스틱 데일리 퀘스트 (Heuristic Daily Quests)

### 1.1. 배경 및 목적
인게임 플레이 중 실시간 정확도(Rate) 수집은 UI 커스텀, 리스크 관리 및 CPU 오버헤드로 인해 지양한다. 대신, 곡 선택이 끝난 정적 상태(`is_stable = true`)에서 저장된 최근 플레이 데이터를 가공하여 개인 맞춤형 미션을 매일 생성하고 달성 여부를 트래킹한다.

### 1.2. 작동 메커니즘 (Heuristics Engine)
* **데이터 수집**: 로컬 DB에서 최근 50~100개의 플레이 기록을 스캔하여 다음 프로필을 도출한다.
  * 선호 버튼 모드 (예: 6B, 8B)
  * 적정 레벨 범위 (예: SC 9 ~ SC 11)
  * 평균 정확도 범위 (예: 99.10% ~ 99.60%)
* **일일 결정적 시드**: 오늘 날짜(`YYYYMMDD`)를 난수 생성기(PRNG)의 시드로 입력하여, 앱을 껐다 켜도 당일에는 항상 동일한 미션셋이 유지되도록 한다.
* **3가지 미션 템플릿**:
  1. **성장형 미션 (Challenger)**: 
     * 대상: 기존 플레이 기록 중 정확도가 특정 목표치(99.00%, 99.50% 등) 바로 직전(예: 99.43%)인 곡.
     * 목표: 해당 곡의 정확도를 목표치 이상으로 갱신.
  2. **탐험형 미션 (Explorer)**:
     * 대상: 보유한 DLC 곡 중 최근 30일간 플레이 이력이 없는 적정 레벨 대역의 곡.
     * 목표: 해당 곡을 플레이하여 98.50% 이상 기록.
  3. **연습량 미션 (Stamina)**:
     * 대상: 선호하는 버튼/난이도 대역.
     * 목표: 당일 평균 정확도 99.20% 이상으로 누적 5판 플레이.

### 1.3. UI & 아트 컨셉 (Minimalist Swiss Style)
* **에셋 프리(Asset-free) 구조**: 외부 디자인 이미지 리소스 없이, 순수 타이포그래피와 레이아웃만으로 구성하여 리스크를 없앤다.
* **컴포넌트 설계**:
  * 얇은 테두리(`stroke: 1px`)와 여유로운 패딩을 사용한 어두운 톤의 카드 레이아웃.
  * 미션 목록 옆에 빈 원(`○`) 표시. 미션 달성 조건 충족 시, 부드러운 전환과 함께 글로우 효과가 포함된 청록색 채워진 원(`●`)으로 상태 변경.
  * 세련된 폰트 비중 조절을 통해 고급 생산성 툴(Linear 등)과 같은 느낌을 준다.

## 2. 깃허브 잔디 스타일 서열표 현황판 (GitHub Grass Style Floor Map)

### 2.1. 배경 및 목적
서열표의 모든 곡들을 무작정 리스트 형태로 보여주는 것보다, 깃허브의 Contribution 잔디밭처럼 시각적 군더더기 없이 깔끔하면서도 정복 욕구를 자극하는 그리드 형태로 시각화한다.

### 2.2. 시각화 아키텍처
* **레이아웃**: 
  * 서열표 레벨(예: SC 10, SC 11)을 가로 줄로 배치하고, 각 줄 내부에는 수십 개의 곡을 아주 작은 둥근 사각형(Cell, `rounding: 2.0`)으로 나열한다.
* **색상 스펙트럼 (정확도 기반)**:
  * 플레이 안 함: 어두운 회색 (배경보다 약간 밝은 색)
  * 98.00% 미만: 차분한 딥 블루 (DJMAX 기본 메인 컬러 계열)
  * 98.00% ~ 99.00%: 중간 톤 블루
  * 99.00% ~ 99.99%: 산뜻한 시안/민트 (목표 달성의 성취감)
  * 100.00% (Perfect Play): 부드럽게 빛나는 느낌을 주는 화이트/골드 얇은 외곽선 하이라이트
* **인터랙션**:
  * 각 셀에 마우스를 올릴 때(Hover), 깔끔한 툴팁이 팝업되며 곡 제목, 최고 기록 정보, 미니 재킷 이미지를 노출한다.

### 2.3. 기대 효과
* 그래픽 리소스를 수작업으로 준비할 필요 없이, `egui` 내부 드로잉 코드로만 표현이 가능하여 깐깐한 아트 기준을 수학적 미니멀리즘으로 만족시킨다.
* 회색 잔디밭을 화려한 민트/골드빛 잔디밭으로 채우고자 하는 강한 수집 욕구를 불러일으킨다.

## 3. 기술적 트레이드오프 및 제한 사항

| 고려 항목 | 설명 | 해결 방안 |
|---|---|---|
| **로컬 DB 쿼리 오버헤드** | 미션 갱신 여부를 판별하기 위해 빈번하게 DB를 쿼리할 경우 성능 저하 우려 | 로컬 메모리상에 `PlayContext` 갱신이 발생할 때만 단발적으로 쿼리하거나 메모리 캐시 활용 |
| **재킷 이미지 로딩** | 4번의 그리드 마우스 호버 툴팁 시 재킷 이미지 메모리 로드 지연 | 필요한 인덱스만 레이지 로딩(Lazy Loading)하도록 렌더링 최적화 |
| **DLC 보유 여부 확인** | 유저가 보유하지 않은 DLC의 곡이 Explorer 미션으로 잡히는 문제 | 로컬 기록 DB에서 최근 플레이 기록을 통해 최소 1회 이상 검출된 DLC 컬렉션 위주로 가중치 부여 |
